## week of may 3, 2026

### new features

- **retention modes** — new media-only mode keeps search alive while reclaiming disk. set per recording type via the `--retention-mode` flag in the [CLI reference](/cli-reference), or use the new one-click "delete last 15 / 30 / 60 minutes" buttons in settings.
- **embedded agent browser** — pi now has its own browser docked as a right-side sidebar. it shows up in **settings → connections** alongside your user browser, with a card for each.
- **Jira via OAuth 2.0** — Jira connections now use 3LO OAuth (no more manual API token). reconnect from **settings → connections**.
- **Supabase integration** — connect Supabase from **settings → connections**. see [connections](/connections).
- **Connections in the sidebar** — Connections has been promoted from a settings sub-page to the main sidebar so your integrations are one click away.
- **tray: pause for a set time** — right-click the tray icon and pick "pause for 5 / 15 / 30 minutes…" and recording auto-resumes on its own. the tooltip shows live state and you'll get a notification when it resumes.
- **queue follow-up prompts** — type your next question while pi is still streaming and it sends as soon as the current turn finishes.
- **one-off scheduled pipes** — schedule a pipe to run at a specific time with `schedule: at <iso>` in the pipe manifest. see [pipes](/pipes).
- **install pipe bundles by URL** — `screenpipe install <manifest-url>` installs a pipe and its dependencies in one shot. see the [CLI reference](/cli-reference).
- **single recording quality knob** — one slider replaces the per-format knobs and fixes unreadable text on ultrawide displays.
- **streaming onboarding summary** — the "here's what i picked up" screen now streams prose with thumbnails as your first capture comes in.
- **Warp MCP connection** — new connection card to wire screenpipe into [Warp's MCP](/mcp-server) clients.
- **help cards** — the help menu now has dedicated cards for documentation and video tutorials.
- **in-app file viewer for notifications** — file links inside notifications now open in screenpipe's viewer instead of a new tab.
- **macOS text events carry `app_name` and `window_title`** — accessibility-tree text events now include the source app and window for better filtering and search.

### updates

- **default model is "auto"** — new chats start on the auto-routed model; stale qwen presets are migrated automatically.
- **chat replies default to plain prose** — pi only switches to the technical/markdown style when your prompt actually warrants it.
- **CLI connection list sorts connected first** — `screenpipe connection list` now puts active connections at the top.
- **pro vs free preset defaults** — first-run defaults are now seeded based on whether you're on a pro plan.
- **CLI desktop-app reminder** — long-running CLI sessions get a periodic, dismissible nudge to install the desktop app. silence it with `SCREENPIPE_NO_REMINDERS=1`.

### bug fixes

- **audio capture** — detects when another app hijacks an input device and feeds zeros, instead of silently recording silence.
- **dark / light mode** — fixed inconsistent theme between Tauri shell and webview, plus full-opacity foreground on dark-mode chat rows.
- **MCP `list-meetings`** — accepts relative dates ("yesterday", "last week") again.
- **MCP `merge-speakers`** — corrected JSON field names so the tool actually merges.
- **AI gateway** — `reasoning_content` is promoted to `content` when the model returns empty content, and model aliases now apply to the request body, not just provider selection.
- **onboarding** — friendly fallback when the summary stream fails, live-feed no longer hangs on quiet engines, prose always renders locally and upgrades to the LLM version if it arrives, and integration cards are now equal height.
- **pipes** — you can now disable a pipe even if its connections are missing; scheduler also skips pipes that are still in setup.
- **owned-browser** — survives tray-only mode and is recognized by `GET /connections/:id`.
- **user-browser openUrl** — falls back to a toast + clipboard copy when the system browser can't be launched.
- **connections panel** — shows loading + retry instead of a blank panel when the API is briefly unreachable; Google Calendar status dot now reflects reality.
- **chat reliability** — user messages persist on send, pipe-watch sessions no longer leak events between chats or strand a "writing…" banner, deleted chats stay deleted, "analyzing…" no longer hangs, and switching presets/model no longer flickers send/stop.
- **sidebar** — "+ new chat" opens a real new thread (and stops jumping selection to old chats), running pipes don't double-render in scheduled, and pipe order is stable.
- **windows** — Escape closes the overlay timeline when opened from search, Escape routes correctly when Home has focus, and the console-flash on `cmd` / `where` spawns is gone.
- **webview title bars** — title text is hidden on the login + Google Calendar OAuth windows (and every other webview).
- **macOS NSPasteboard** — clipboard reads are now serialized on a dedicated dispatch queue, fixing rare crashes during clipboard capture.
- **browser extension** — picks eligible tabs (skipping `chrome-extension://` pages), keepalive every 30s, and re-tries the owned-browser install on failure.
- **tray** — transient `/health` flaps no longer flip the tray to "Error".
- **database** — write-queue now treats SQLite I/O and corruption errors as fatal so they surface immediately instead of silently dropping writes.
- **keychain / encryption** — non-Mac platforms gate cleanly through onboarding; secret-encryption requests are unified across the app.

## week of april 26, 2026

### new features

- **new integrations** — connect [Microsoft 365 and Teams](/connections), Google Docs, Google Sheets, QuickBooks Online, Bitrix24, Loops, Resend, Supabase, and the Pocket AI voice recorder. configure in **settings → connections**.
- **PII privacy filter** — opt-in filter (powered by a Tinfoil enclave) that redacts personal info from `/search` results and chat messages before they leave your machine. toggle from the chat composer. [learn more](/privacy-filter).
- **parallel chats with background streaming** — start multiple chats and switch between them; responses keep streaming in the background. a new chat sidebar shows pinned chats, recents, and pipes scheduled to run.
- **focus-aware capture** — when enabled, screenpipe only records the focused monitor instead of all of them, halving capture cost on multi-display setups. toggle from settings.
- **per-machine pipe favorites** — star pipes you use most. starred pipes sort to the top of the pipe list and get their own filter chip.
- **LAN-access toggle for the API** — bind the local API to `0.0.0.0` so other devices on your network can reach it. API auth is force-enabled when LAN access is on. see the [API docs](/for-developers).
- **`screenpipe sync remote`** — back up your screenpipe data over SSH/SFTP to a server you control. see the [CLI reference](/cli-reference).
- **`screenpipe db {check, recover, cleanup, unlock}`** — new CLI commands to repair database corruption and reclaim disk space. see the [CLI reference](/cli-reference).
- **`screenpipe logout`** — sign out of your screenpipe cloud account from the CLI.
- **connected apps as @mentions in chat** — type `@` in the chat filter popover to scope a question to a specific connected integration.
- **AI quota warnings** — heads-up notice when you're close to your daily limit on weighted models, so you don't get cut off mid-task.
- **browser extension v0.2** — new popup with connection status, an options page, and token-based auth.

### updates

- **CoreAudio Process Tap is now the default for system audio** on macOS 14.4 and later — better quality and no virtual driver required.
- **API keys are now readable and regeneratable** in the UI. user-set custom keys have been removed in favor of a single managed key.
- **Cursor-style inline edit** — click any of your past chat messages to edit and re-run it.
- **redesigned sidebar** — collapse button next to the macOS traffic lights, search opens a focused overlay window with a real keyboard shortcut, and starred pipes outrank running pipes in sort order.
- **timeline calendar** — empty days are now disabled in the day picker and skipped when navigating with arrow keys.
- **chat models hot-swap** — switching a preset's model no longer restarts the chat subprocess.

### bug fixes

- **mic stays connected on sleep/wake** — fixed false-positive disconnects.
- **Microsoft 365 / Teams OAuth** now uses the correct Azure client ID; personal Microsoft accounts are rejected up-front (Teams scope isn't supported there).
- **database reliability** — multibyte string truncation no longer panics, and connection starvation under heavy writes has been resolved.
- **diarization timeouts** — added a `speaker_id` index that fixes 60s+ stalls on long meetings.
- **memories pipeline** silently stopped updating — now back online.
- **macOS memory leaks** — wrapped clipboard capture, monitor enumeration, and focus-tracker callbacks in autorelease pools.
- **search results** — keyword flat mode no longer drops entries that lack text positions.
- **OAuth flows** — auto-refresh of expired tokens in the generic proxy, query params now forwarded through the connection proxy, broader Google Sheets scope, and a fix for tokens being dropped when reconnecting an instance.
- **notifications** — clicking "Open" on a native macOS notification now reliably brings the screenpipe window forward.
- **onboarding** — MCP install uses the bundled bun, so it works without a system bun on `PATH`. WhatsApp connection now resolves bun the same way.
- **calendar permission** — recovers when the macOS Privacy pane reports denied-but-empty.
- **chat polish** — drag-select restored in user messages, "+ new chat" reuses an empty chat instead of spawning duplicates, "Try again" appears when a model returns an empty response, and free models are always allowed regardless of quota.
- **Windows** — restored ARM64 release builds, switched consumer release to SSL.com EV signing, and propagated system root CAs to bundled bun/node via `NODE_EXTRA_CA_CERTS`.
